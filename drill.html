ドリル課題まとめ

Claude Code 研修で取り組んだこと

TOKIUM入社研修で実施した Claude Code ドリル課題（全20問・7セット）の学びをまとめました。

Day 1

必須セット1：基本操作

CC#1 はじめの一歩

ターミナルで claude と打って Claude Code を起動。「自分の名前と TOKIUM 26卒 が表示されるページを作って」とお願いすると、HTMLファイルを自動で作ってブラウザに表示してくれた！

日本語で「こんなの作って」と話しかけるだけでOK。ファイルを作るときはちゃんと「作っていい？」と聞いてくれるので安心。

CC#2 許可の仕組みを理解する

「新しいファイルを作って」と言うとファイル作成の許可が、「ブラウザで開いて」と言うとコマンド実行の許可が出てくる。同じ「許可」でも種類が違うことを発見！ y で今回だけOK、n でやめて、a でずっとOK、と選べる。

何でもOKを押すのではなく、「今なにを許可しようとしてるかな？」と中身を見る習慣が大切。お仕事でも同じ！

CC#3 指示の出し方で結果が変わる

たとえば「自己紹介を追加して」だけだと Claude の好みで作られちゃう。でも「名前・出身地・趣味3つを箇条書きで、背景は水色で追加して」と伝えると、イメージ通りのものが出てくる。同じことをお願いしてるのに、伝え方で全然違う！

長く書けばいいわけじゃなくて、「何がほしいか」のポイントを押さえるのがコツ。

CC#4 ファイルを読んで直す

「自己紹介のファイルを読んで」と言えば中身を見てくれるし、「文字をもっと大きくして」「背景を水色にして」と見た目の言葉で修正できる。さらに社員データの入った sample.txt を渡して「見やすい表にして」とお願いすると、きれいな表に変換してくれた。

ゼロから作るだけじゃなくて、今あるものを読んで直すのも得意。お仕事では「直す」場面のほうが多いから、この使い方が一番役立ちそう。

必須セット2：CLAUDE.md

CC#5 CLAUDE.mdを書く

CLAUDE.md は「Claude への指示書」。たとえば「メインカラーは青系」「日本語で書いて」「砂浜っぽい雰囲気で」と書いておくと、新しいページを作るたびに毎回言わなくても自動でそのルールに従ってくれる。

毎回同じ説明をしなくていいのがすごく楽！ 書く内容が具体的なほど、出力のクオリティが安定する。

CC#6 セッション管理

会話が長くなると Claude Code の動きが遅くなることがある。そんなときは /compact と打つと会話を圧縮してくれる。終了後も claude --continue で前回の続きから再開できるので、翌日もスムーズ。

「なんか重いな」と感じたら /compact。作業の区切りで終了しても、次の日ちゃんと続きから始められるので安心。

必須セット3：設計して作る

CC#7 計画してから作る

/plan と打つと計画モードになって、「こういう構成でいこうと思うけどどう？」と Claude が提案してくれる。「ここはこうしたい」と意見を言いながら一緒に計画を詰めて、OKなら「この計画で進めて」で一気に作ってくれる。

いきなり作り始めると「やっぱ違う…」の手戻りが起きがち。先に計画を立てるだけで完成度がぐっと上がる！

CC#8 壊れたコードを直す

表示が崩れた HTML ファイルを渡して「何がおかしいか調べて。まだ直さないで」とお願いすると、原因を説明してくれる。難しかったら「もっと簡単に説明して」と言えばOK。納得してから「じゃあ直して」で修正。

「直して」の一言でも直してくれるけど、「何が壊れてたのか」を自分でもわかっておくのが大事。エラーが出たらそのまま貼るのが最速の解決法！

CC#9 複数ファイルで構成する

「ポートフォリオを3つのファイルに分けて」と指示すると、HTML（中身）・CSS（見た目）・JS（動き）に分けてくれる。分けた後に「背景色を変えて」とお願いすると、CSSだけ変わってHTMLはそのまま。便利！

1つのファイルに全部書くと長くてごちゃごちゃに。役割ごとに分けると「見た目だけ直したい」がサッとできる。

任意セット4：外部連携

CC#10 Slackに投稿する

「研修チャンネルに『CC#10完了！』と投稿して」とお願いすると、Claude Code が Slack にメッセージを送ってくれる。送る前に「この内容でいい？」と確認画面が出るので、中身を見てからOKする。

PC外のサービス（Slack など）も操作できるのはすごいけど、その分「何を送ろうとしてるか」をちゃんと見る習慣が大事。

CC#11 ブラウザで情報を取る

「https://tokium.jp にアクセスして会社の概要をまとめて」と言うと、Webサイトの情報を読み取ってまとめてくれる。それを「ポートフォリオに追加して」と言えば、新しいセクションとして組み込んでくれた。

調べものもお任せできるけど、AIがまとめた情報が正しいかは自分の目で確認するのが鉄則！

CC#12 Driveにファイルを保存する

「portfolio フォルダの中身を Google Drive にコピーして」と言うだけでバックアップ完了。PCが壊れても安心だし、URLで人に共有もできる。

ファイル操作もSlack投稿もブラウザも Drive も、ぜんぶ Claude Code の画面1つでできちゃう。

Day 2

必須セット5：CLAUDE.md 深掘り

CC#13 CLAUDE.mdの威力を実感する

CLAUDE.md をいったん無効にして「趣味のページを追加して」と指示 → デザインがバラバラに。CLAUDE.md を戻してもう一度同じ指示 → ちゃんと統一されたデザインで出てくる。この差がはっきりわかって面白かった！

CLAUDE.md があるだけで「毎回言わなくても」品質が安定する。ないと、全部イチから説明しないといけなくなる。

CC#14 Claude Codeにレビューしてもらう

「スマホで見たとき気になるところを3つ教えて」のように聞くと、改善ポイントを具体的に教えてくれる。「確かに！」と思ったものだけ選んで直してもらい、気に入らなければ「元に戻して」で戻せる。

AI は「作る」だけじゃなくて「チェックする」のも得意。ただし全部の指摘に従う必要はなくて、「どれを直すか決めるのは自分」というのが大事。

CC#15 CLAUDE.mdを育てる

CC#14 で「フォントサイズがバラバラ」と指摘されたなら、それを CLAUDE.md に「本文は 0.92rem に統一」と追記する。すると次からは自動で守ってくれるようになる。

CLAUDE.md は一度書いて終わりじゃなくて、「気づいたことを足していく」もの。この「気づき → ルール化 → 自動化」のサイクルは、お仕事の改善と同じ考え方！

必須セット6：Skills

CC#16 繰り返し作業をSkillにする

毎回「CLAUDE.md のデザインに合わせて新しいページを作って、ブラウザで開いて」と言うのが面倒…。そこで、この一連の手順を Skill ファイルにまとめておくと、次からは /add-page と打つだけで全部やってくれる！

CLAUDE.md は「常に効くルール」、Skill は「呼び出した時だけ動く手順書」。「面倒だな」と思ったら Skill 化のチャンス。

CC#17 Skillを改善して使い回す

CC#16 で「趣味ページ」用に作った Skill を「好きな食べ物ページ」でも使ってみる。うまくいかない部分があったら「Skill の中身を見せて」→「ここをこう変えて」で改善。2回目・3回目とどんどん使いやすくなっていく。

最初から完璧な Skill は作れなくて当たり前。「使う → 直す → また使う」を繰り返して育てていくのがコツ。

任意セット7：組み合わせ

CC#18 Slack投稿Skillを作る

CC#10 の Slack 投稿と CC#16 の Skill を組み合わせ！ /progress セット7に挑戦中です と打つだけで、テンプレート付きの進捗報告が Slack に投稿される Skill を作った。

「外部連携 × 自動化」の組み合わせが一番パワフル。日報の投稿や定型の連絡など、お仕事の繰り返し作業も同じやり方で効率化できそう。

CC#19 ポートフォリオを磨き上げる

「スマホサイズで表示が崩れないか確認して」「色やフォントがバラバラなところを見つけて」など、5つの観点で Claude Code にチェックしてもらう。問題が見つかったら「直して」→ ブラウザで確認、の繰り返し。

「作って終わり」じゃなくて「使う人の目線で見直す」のが大事。チェックリストを渡して Claude Code に確認してもらうのは、品質を上げるいい方法！

CC#20 CLAUDE.md + 一発で作らせる

2日間かけて育てた CLAUDE.md の集大成。「ダークモード切替機能を追加して」のように1回だけ指示を出して、追加の説明なしでどこまで完成するかチャレンジ。イマイチだったら、指示を長くするんじゃなくて CLAUDE.md のほうを改善して再挑戦！

「1回で伝わるように設計する力」こそが、AIを使いこなすスキル。指示を長くするより、CLAUDE.md を育てるほうが近道！
